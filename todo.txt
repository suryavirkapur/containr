how the flow rn differs from render:
- znskr combines everything into a single massive form (`newapp.tsx`). you pick source (git or template), then runtime shape (web vs private), then pick the repository, then fill out the environment variables and service definitions all on the exact same page.
- znskr's dashboard has cards acting as "launch paths" that link to `/services/new` with query parameters.
- render uses a step-by-step funnel. 
  1. you click a "new" dropdown menu in the navigation bar and select the target service type (e.g. "web service").
  2. for git services, you land on a dedicated "connect a repository" page showing only the github repos or a public url input.
  3. after picking the repo, you are navigated to a dedicated configuration page displaying a linear, single-column form (name, branch, root directory, runtime, build command, start command, instance type, env vars) to submit and create.
  4. template services (postgres/redis) have their own isolated, hyper-focused linear configuration pages.
