installation
============

containr is a rust-native paas for deploying docker containers with automatic ssl.

requirements
------------

- linux x86_64
- rust 1.75+ (install via https://rustup.rs)
- docker (containerd runtime)
- mise
- node 24 (via mise, for frontend)
- pnpm (via mise, for frontend)
- openssl (for generating secrets)

quick install
-------------

  curl -fsSL https://github.com/suryavirkapur/containr/releases/latest/download/containr-install-vps.sh | sudo bash

detailed installation
---------------------

option 1: install from cargo (recommended)

  cargo install --git https://github.com/suryavirkapur/containr --package containr --bin containr
  cargo install --git https://github.com/suryavirkapur/containr --package containrctl --bin containrctl

with specific version/tag:

  cargo install --git https://github.com/suryavirkapur/containr --tag v0.1.15-alpha --package containr --bin containr
  cargo install --git https://github.com/suryavirkapur/containr --tag v0.1.15-alpha --package containrctl --bin containrctl

option 2: build from source

  git clone https://github.com/suryavirkapur/containr.git
  cd containr
  cargo build --release
  sudo cp target/release/containr /usr/local/bin/

option 3: install to cargo bin

  git clone https://github.com/suryavirkapur/containr.git
  cd containr
  cargo install --path crates/containr
  cargo install --path crates/containrctl

post-install configuration
--------------------------

option 1: use install script (recommended)

  ./install.sh

this creates containr.toml with randomized secrets and creates ./data/certs directory.

option 2: create config manually

  cat > containr.toml << 'EOF'
  [server]
  host = "0.0.0.0"
  port = 2077

  [database]
  path = "./data/containr.sqlite3"

  [cache]
  path = "./data/cache"

  [logging]
  dir = "./data/logs"
  retention_days = 14

  [proxy]
  http_port = 80
  https_port = 443
  base_domain = "your-domain.com"
  public_ip = "203.0.113.10"

  [github]
  client_id = ""
  client_secret = ""
  webhook_secret = ""

  [auth]
  jwt_secret = "$(openssl rand -hex 32)"
  jwt_expiry_hours = 24

  [security]
  encryption_key = "$(openssl rand -hex 32)"

  [acme]
  email = "you@example.com"
  certs_dir = "./data/certs"
  staging = true

  [storage]
  data_dir = "/data/containr"
  max_volume_size_gb = 10
  backup_enabled = false
  rustfs_endpoint = "http://localhost:9000"
  rustfs_access_key = ""
  rustfs_secret_key = ""
  EOF

  mkdir -p ./data/certs

running
-------

with default ports (requires root for 80/443):

  sudo containr

or explicitly:

  sudo containr server

with custom ports (no root required):

  containr --http-port 8080 --https-port 8443 --api-port 2077

command line options:

  containr server
      run the full api/proxy/worker stack

  containr docker check
      verify docker socket connectivity and print engine details

  containr docker containers --all
      list containers directly from the docker socket

  containr generate-api-key --email you@example.com --raw
      create a long-lived bearer api key for an existing user
      note: with the default sled backend, stop containr first so the database lock is free

  containr setup-systemd
      render a systemd service unit file, optionally writing it to /etc/systemd/system

client cli
----------

containr now ships a separate client binary:

  containrctl init --url http://127.0.0.1:2077 --instance-id local
  containrctl register --email you@example.com --password your-password
  containrctl health
  containrctl projects list
  containrctl containers list
  containrctl system stats

client config is stored in:

  $HOME/.containr/config.toml

the config keeps the active instance, instance id, base url, bearer token/api key,
timeout, and tls verification settings.

frontend development
--------------------

the frontend is optional for deployment but required for development:

  mise install
  cd web
  pnpm install
  pnpm dev

for production build:

  mise install
  cd web
  pnpm build

systemd service
---------------

generate a unit file automatically:

  sudo containr setup-systemd \
    --binary-path /usr/local/bin/containr \
    --working-directory /opt/containr \
    --config-path /opt/containr/containr.toml \
    --enable \
    --start

or create /etc/systemd/system/containr.service manually:

  [unit]
  description=containr paas
  after=network.target docker.service
  requires=docker.service

  [service]
  type=simple
  user=root
  workingdirectory=/opt/containr
  execstart=/usr/local/bin/containr
  restart=always
  restartsec=5
  environment=rust_log=info

  [install]
  wantedby=multi-user.target

enable and start:

  sudo systemctl daemon-reload
  sudo systemctl enable containr
  sudo systemctl start containr

view logs:

  sudo journalctl -u containr -f

docker install (alternative)
----------------------------

build docker image:

  docker build -t containr .

run with docker:

  docker run -d \
    --name containr \
    --restart unless-stopped \
    -p 80:80 \
    -p 443:443 \
    -p 2077:2077 \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -v $(pwd)/data:/app/data \
    -v $(pwd)/containr.toml:/app/containr.toml:ro \
    containr

docker compose:

  version: '3.8'
  services:
    containr:
      build: .
      restart: unless-stopped
      ports:
        - "80:80"
        - "443:443"
        - "2077:2077"
      volumes:
        - /var/run/docker.sock:/var/run/docker.sock
        - ./data:/app/data
        - ./containr.toml:/app/containr.toml:ro

verify installation
-------------------

check version:

  containr --version

check health:

  curl http://localhost:2077/health

expected response:

  {"status":"ok"}

access dashboard:

  open http://localhost:2077

dns setup
---------

point your base domain and wildcard to your server:

  your-domain.com     A     your-server-ip
  *.your-domain.com   A     your-server-ip

or for testing with /etc/hosts:

  127.0.0.1  your-domain.com
  127.0.0.1  app1.your-domain.com
  127.0.0.1  app2.your-domain.com

github oauth setup
------------------

1. go to github settings > developer settings > oauth apps
2. create new oauth app:
   - application name: containr
   - homepage url: https://your-domain.com
   - callback url: https://your-domain.com/api/auth/github/callback
3. copy client id and client secret to containr.toml

github webhooks setup
---------------------

for automatic deployments on push:

1. go to your repo settings > webhooks
2. add webhook:
   - payload url: https://your-domain.com/webhooks/github
   - content type: application/json
   - secret: same as webhook_secret in containr.toml
   - events: just the push event
3. test with a push to your repo

troubleshooting
---------------

permission denied on port 80/443:
  run with sudo or use custom ports:
    containr --http-port 8080 --https-port 8443

docker not found:
  install docker and add user to docker group:
    sudo usermod -aG docker $USER

config file not found:
  run install.sh or create containr.toml manually

ssl certificate errors:
  set staging = true for testing
  ensure dns is configured correctly
  check acme email is valid

database errors:
  ensure ./data directory exists and is writable

sqlite is now the default backend for new installs.
if you intentionally use sled, commands that open the database directly may require the service to be stopped first.

more help:
  see docs/troubleshooting.txt
