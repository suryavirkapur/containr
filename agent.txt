important decisions:

- app credentials (private key, secrets) are stored encrypted in sled
- acme issuance is triggered on first https request and deduped per domain
- acme requires config.acme.email to be set, otherwise it is disabled

architectural notes:
- github app auth uses jwt for app level and installation tokens for repo level
- used sled for storing github app configuration and installations
- acme manager runs on tokio runtime and serves http-01 via shared challenge store
- pingora tls callback only queues issuance to avoid blocking handshake path
- upstream resolution errors return a proxy error instead of panicking
- api upstream uses loopback when config host is 0.0.0.0 or ::
