changelog
=========

0.1.6-alpha - 2026-01-17
- add deployment logs modal to appdetail page
- add logs button to each deployment row with historical and live streaming
- add reissue certificate button to settings page
- create bucketdetail page with connection info and copy buttons
- add /storage/:id route for bucket details
- make bucket names clickable links in storage list
- improve queuedetail page with credentials section and copy buttons

0.1.5-alpha - 2026-01-17
- migrate to bollard 0.20 api for docker operations
- replace sync docker cli calls with async bollard client
- update all container/network/image operations to use bollard::query_parameters
- fix option types and generics for bollard 0.20 compatibility

0.1.4-alpha - 2026-01-17
- persist acme account credentials to avoid let's encrypt rate-limiting
- account credentials saved as account.json and reused across restarts

0.1.3-alpha - 2026-01-16
- auto-generate ssl certificates when app is created/updated with a domain
- add cert_request_tx channel to api state for proactive certificate issuance

0.1.2-alpha - 2026-01-16
- fix missing content-length headers in pingora proxy responses
- browsers now correctly load static files, acme challenges, and error pages

0.1.1-alpha - 2026-01-15
- trigger acme issuance when tls certs are missing
- add acme request worker with in-flight dedupe
- avoid upstream resolution panics in proxy
- route base domain api requests to the api server

0.1.0-alpha - 2026-01-15
------------------------

initial alpha release

core features:
- docker container deployment from github repositories
- automatic containerfile/dockerfile detection and building
- pingora reverse proxy with http/https routing
- acme/let's encrypt automatic ssl certificates
- sled embedded database for persistent state

protocol support:
- websocket passthrough for real-time applications
- grpc (http/2) passthrough support
- server-sent events (sse) streaming support

multi-container apps:
- container services with replica scaling
- service dependencies (depends_on)
- health checks with configurable intervals
- restart policies (never, always, onfailure)
- memory and cpu limits per service

environment and secrets:
- environment variables per app
- secret masking (values hidden in api responses)
- encrypted storage for sensitive values

managed services:
- managed databases (postgres, mysql, redis)
- managed queues (redis, rabbitmq)
- storage buckets

dashboard:
- solidjs dashboard with dark theme
- real-time container logs via websocket
- deployment history and status
- certificate management ui

authentication:
- jwt-based authentication
- github oauth integration
- user registration and login

api:
- restful api for all operations
- openapi documentation at /api/docs
- webhook support for github push events

documentation:
- docs/quickstart.txt: 5-minute quickstart guide
- docs/architecture.txt: system architecture overview
- docs/api-reference.txt: complete api documentation
- docs/configuration.txt: detailed config options
- docs/deployment-guide.txt: production deployment guide
- docs/troubleshooting.txt: common issues and solutions

known limitations:
- linux x86_64 only
- requires root for ports 80/443 (or use custom ports)
- acme staging mode recommended for testing
